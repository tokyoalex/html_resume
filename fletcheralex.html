UK / Czech Game Executive (Over 3 years)

The "Triple 7" touchscreen gaming cabinet is a bespoke fat client embedded Windows XPE application that runs on Barcrests custom made hardware. The gaming system allows both in house and third party games to run on the cabinets via a common API, and also allows multiple video games to be run on each cabinet such that the player can dynamically select a game. The Game Executive application handles processes such as cash in / cash out events, switch events, accountancy, game process management, secure non volatile storage, hardware management and software configuration (e.g. games and themes installed). The main application is multi-threaded, is split into several modules, and communicates with games, themes and several remote services over TCP / UDP; it also communicates with a proprietary hardware controller over USB through a C based API. Persistent configuration data for the application is spread over local flat files, XML files, non volatile RAM, and on a SQL database server. The touch-screen GUI is based on an SDL / Open GL wrapper, although the application supports a web interface, allowing all configuration to be carried out remotely.

Portrait Software

The Portrait development platform allows programmers to rapidly implement business logic and processes using predefined functionality generic to all CRM solutions. It is deployed as a highly scalable, multi-tier, thin client system.
Part of the Portrait development suite is a set of GUI configuration tools; that allow a user to graphically model business processes, with unlimited layers of complexity (i.e. models contain models). The configuration output is saved in as xml. Following validation it's deployed to a MS-SQL database where the configuration is retrieved, parsed and executes specified componentised binaries (as COM or .Net Assembly dlls) at runtime.
Parts of the configured logic also define the interface displayed via the client (HTA) application in the form of aspx pages with Portrait / .Net web controls. The controls are populated from web service calls, which access server side components that amongst other things call stored procedures, or adaptors that allow access to legacy systems.
The 3r application is the predecessor to the Portrait Application Suite and is a similar concept, except that it has a fat client architecture and there are no GUI tools, but it too allows clients projects to build on a predefined CRM architecture and standardised / customisable MFC interface. The application was written using Microsoft Visual C++.
